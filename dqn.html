PART 01

The Foundation — Q-Values

The MDP Framework

Every Q-value lives inside a Markov Decision Process. The agent exists in a world described by five components:

Symbol Name Meaning
s State Where the agent is / what it observes
a Action What the agent can do at each step
r Reward Scalar feedback signal from the environment
s' Next State State after taking action a in state s
π Policy Strategy mapping states to actions
Defining the Q-Value

The Q-value (action-value function) answers one precise question:

"If I'm in state s, take action a, and follow policy π forever — what is my expected cumulative reward?"
Q-VALUE DEFINITION Qπ(s, a) = 𝔼π [ Σt=0∞ γt · rt | s0=s, a0=a ]
Symbol Meaning
γ (gamma) Discount factor (0–1): encodes that future rewards are worth less than immediate ones
rt Reward received at timestep t
𝔼π Expected value when following policy π
PART 02

Deriving the Bellman Equation

This is the core derivation. We decompose the infinite sum into a recursive structure.

1
Expand the infinite sum
Qπ(s,a) = 𝔼π [ r0 + γr1 + γ²r2 + ··· | s0=s, a0=a ]
2
Factor out the first reward
= 𝔼π [ r0 + γ · (r1 + γr2 + ···) | s0=s, a0=a ]
3
Recognize the recursive structure

The term in parentheses is the return from the next state s'. Replace it:

= 𝔼π [ r0 + γ · Vπ(s') ] where Vπ(s') = 𝔼a'~π[Qπ(s',a')]
4
Bellman Equation for Q
Qπ(s,a) = 𝔼s' [ r + γ · Σa' π(a'|s') · Qπ(s',a') ]

Reading this: Q-value now = immediate reward + discounted expected Q-value of next state under policy π

PART 03

Q-Learning & The Scalability Problem

Optimal Bellman Equation

We don't want any policy — we want the optimal one. When acting greedily (always choosing the best next action):

OPTIMAL BELLMAN (Q*) Q*(s,a) = 𝔼s' [ r + γ · maxa' Q*(s',a') ]

The optimal policy then becomes trivially: π*(s) = argmaxa Q*(s,a)

Tabular Q-Learning Update Rule

In classical Q-learning we store Q-values in a table and refine them with every experience:

TD UPDATE RULE Q(s,a) ← Q(s,a) + α · [ r + γ · maxa' Q(s',a') − Q(s,a) ]
Term Name Meaning
α Learning rate How quickly to update the estimate
r + γ·max Q(s',a') TD Target What Q should be based on new evidence
Q(s,a) Current estimate What we currently believe Q is
δ = TD Target − Q(s,a) TD Error The "surprise" — how wrong we were
Why Tabular Q-Learning Breaks
Problem: Atari Pong state = raw pixels = 210 × 160 × 3 ≈ 100,000 dimensions. The number of unique states is astronomically large — you cannot have a table row for each one. We need to generalize across states.

This is the precise motivation for approximating Q with a neural network.

PART 04

DQN Neural Network Architecture

Core Idea: Replace the Table

Parameterize Q with a neural network Q(s, a; θ). The network takes a state as input and outputs Q-values for all actions simultaneously in a single forward pass.

Atari DQN Architecture (DeepMind 2015)
DQN Architecture — Atari
Input: 4 stacked grayscale frames (84 × 84 × 4) │ ▼ ┌─────────────────────────────────────────────────────┐ │ Conv Layer 1 │ 32 filters, 8×8 kernel, stride 4 │ → (20 × 20 × 32) │ Activation: ReLU │ └─────────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ Conv Layer 2 │ 64 filters, 4×4 kernel, stride 2 │ → (9 × 9 × 64) │ Activation: ReLU │ └─────────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ Conv Layer 3 │ 64 filters, 3×3 kernel, stride 1 │ → (7 × 7 × 64) │ Activation: ReLU │ └─────────────────────────────────────────────────────┘ │ ▼ Flatten → 3136 units │ ┌─────────────────────────────────────────────────────┐ │ Fully Connected │ 512 units │ │ Activation: ReLU │ └─────────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ Output Layer │ N units (one Q-value per action) │ │ No activation (raw Q-values, can be negative) │ └─────────────────────────────────────────────────────┘
Design Choices Explained
Choice Reason
Convolutional layers State is an image — CNNs automatically extract spatial features (edges, shapes, motion blur) without hand-engineering
4 stacked frames A single frame has no velocity. 4 consecutive frames give the network a sense of direction and speed (e.g., which way the ball is moving in Pong)
Grayscale + 84×84 Reduces input dimensionality significantly while preserving sufficient visual information
ReLU activations Avoids vanishing gradients; computationally cheap; works well with convolutional networks
No output activation Q-values are unbounded real numbers — a sigmoid or softmax would incorrectly constrain them
Output: one node per action Computes Q(s,a) for all actions in one forward pass — vastly more efficient than separate forward passes per action
PART 05

The Two Critical DQN Innovations

Naively plugging a neural net into Q-learning diverges catastrophically. DeepMind solved this with two breakthroughs:

🔁
Experience Replay
Problem: Correlated data

Consecutive frames are nearly identical. Training on them sequentially causes the network to overfit recent experience and catastrophically forget the past.


Solution: Store transitions (s, a, r, s') in a Replay Buffer of ~1M entries. At each step, sample a random mini-batch. This breaks correlation and allows each experience to be reused multiple times.

🎯
Target Network
Problem: Moving targets

The TD target uses the same network being updated. Both prediction and target shift every step — like chasing a moving target. This causes oscillation and divergence.


Solution: Keep a separate frozen target network θ⁻. Update it only every C steps (e.g., 10,000). The main network trains against this stable reference.

Target Network — Two Networks Compared
Network Role Updated How?
Online Network θ Makes predictions, selects actions Every step via gradient descent
Target Network θ⁻ Computes TD targets Frozen; copied from θ every C steps
DQN LOSS FUNCTION ℒ(θ) = 𝔼(s,a,r,s')~D [ ( r + γ·maxa' Q(s',a'; θ⁻) − Q(s,a; θ) )² ]

Note: θ⁻ is frozen during the gradient step. Only θ (online network) receives the gradient.

PART 06

Full DQN Algorithm

DQN — Complete Pseudocode
INITIALIZE: Online network Q(s,a; θ) with random weights θ Target network Q(s,a; θ⁻) with θ⁻ ← θ Replay buffer D (capacity N ≈ 1,000,000) ε = 1.0 // start fully random (explore) FOR EACH episode: s ← initial state // preprocess: grayscale → resize → stack 4 frames FOR EACH timestep t: ── ACTION SELECTION (ε-greedy) ───────────────────────── if random() < ε: a ← random_action() // explore else: a ← argmaxa Q(s, a; θ) // exploit ── ENVIRONMENT STEP ──────────────────────────────────── r, s' ← env.step(a) push(D, (s, a, r, s')) // store in replay buffer s ← s' ── SAMPLE & TRAIN ────────────────────────────────────── batch ← random_sample(D, size=32) for each (s, a, r, s') in batch: if s' is terminal: y = r else: y = r + γ · maxa' Q(s', a'; θ⁻) // target network! loss = ( y − Q(s, a; θ) )² gradient_descent(loss, θ) ── TARGET SYNC ───────────────────────────────────────── if t % C == 0: θ⁻ ← θ // copy weights every C steps ── EPSILON DECAY ─────────────────────────────────────── ε ← max(0.1, ε − decay_rate) // anneal from 1.0 → 0.1

Follow-up Question — Deepen Your Understanding

DQN's output head is a simple linear layer producing Q(s,a) for every action from a single forward pass. A later improvement called Dueling DQN splits this head into two separate streams:

Stream Estimates Notation
Value stream How good is this state, regardless of action? V(s)
Advantage stream How much better is action a vs. average? A(s,a)

The two are combined as: Q(s,a) = V(s) + A(s,a) − meanaA(s,a)

"Why might separating V(s) and A(s,a) be beneficial? In what kind of states would knowing V(s) alone — without caring which specific action to take — be most useful?"

Thinking through this question will lead you naturally into understanding Dueling DQN and the concept of advantage functions — the next step in the DQN evolution.
