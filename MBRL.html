The Core Idea: What Is a Model?

In standard (model-free) RL, an agent learns a policy π(a|s) purely by trial-and-error interaction with the environment — it never explicitly reasons about what will happen next. Model-based RL (MBRL) changes this: the agent also maintains a learned dynamics model that approximates how the environment transitions.

Formal Definition: Environment Model

A model M is a learned approximation of the true environment dynamics. It typically consists of a transition model and a reward model:

// Transition model p̂_θ(s_{t+1} | s_t, a_t) ≈ p(s_{t+1} | s_t, a_t) // true env dynamics // Reward model r̂_θ(s_t, a_t) ≈ r(s_t, a_t) // With model, we can generate synthetic rollouts: ŝ_{t+1} ~ p̂_θ(· | s_t, a_t) r̂_t = r̂_θ(s_t, a_t)

Once a model is learned, the agent can plan inside it — generating hundreds of imagined trajectories at almost zero cost (no real-world interaction), extracting policy gradients from them, or using them to supplement the real-data replay buffer. This is the mechanism that makes MBRL dramatically more sample efficient than model-free counterparts.

Two Roles a Model Can Play

Model for Planning
The model is used at decision time to simulate future trajectories and select the best action via MPC (Model Predictive Control) or tree search. Examples: PETS, MPPI.
MPC Planning No Policy Network
Model for Learning
The model generates synthetic data (imagined rollouts) that are used to train a separate policy/value function. Examples: Dreamer, MBPO, TD-MPC2.
Dyna-style Latent Space Imagination

In model-free RL, the agent must observe a real transition to learn from it. But in Dyna-style MBRL, synthetic transitions are generated from the model and mixed with real data. World Models: Learning to Dream

A World Model is a specific class of dynamics model that learns a compact latent representation of the environment state and predicts future states, rewards, and terminals entirely within this compressed latent space — enabling the agent to "imagine" and train without ever stepping into the real environment.

World Model vs. Simple Dynamics Model

A naive dynamics model predicts next states in raw observation space (pixels → pixels), which is computationally prohibitive. A world model instead encodes observations into a compact stochastic latent variable z_t, then predicts future latent states — predictions are cheap and can run at scale.

The Recurrent State-Space Model (RSSM) — Heart of DreamerV3

DreamerV3's world model is a Recurrent State-Space Model. It maintains a hybrid state combining a stochastic component z_t (for expressiveness) and a deterministic component h_t (for temporal memory). This enables it to model environments with partial observability and long temporal dependencies.

// RSSM equations h_t = GRU(h_{t-1}, z_{t-1}, a_{t-1}) // deterministic recurrence z_t ~ q_φ(z_t | h_t, o_t) // posterior: uses real obs o_t z_t ~ p_φ(z_t | h_t) // prior: used during imagination // World model objective (ELBO) L_WM = E[ Σ_t β_pred · L_pred(o_t, r_t, c_t) // prediction loss + β_dyn · KL[q_φ||p_φ] // dynamics KL + β_rep · KL[q_φ||sg(p_φ)] ] // representation KL

Imagination-Based Policy Optimization

Once the RSSM is trained, the actor and critic are trained entirely within imagination: imagined trajectories of length H are rolled out from the prior, and gradients flow back through the world model dynamics. No real environment interaction is needed for policy learning.

Major MBRL Algorithms

PETS
Chua et al., 2018 · NeurIPS
Probabilistic Ensembles with Trajectory Sampling. Trains an ensemble of neural network dynamics models (each a Gaussian outputting mean + variance). Uses CEM (Cross-Entropy Method) for planning. Pure model-for-planning — no separate policy network.
Ensemble CEM Planning Uncertainty
MBPO
Janner et al., 2019 · NeurIPS
Model-Based Policy Optimization. Combines short model rollouts (H=1–5) from an ensemble with a SAC policy trained on a mixed real+synthetic buffer. Key insight: short rollouts avoid compounding error.
Dyna-style SAC policy Mixed buffer
DreamerV3
Hafner et al., 2023 · Nature 2025
World model with RSSM + imagination-based actor-critic. Works across diverse tasks (pixels, vectors, continuous/discrete) with fixed hyperparameters. First to master Minecraft diamond collection from scratch.
World Model RSSM Universal
TD-MPC2
Hansen et al., 2024 · ICLR
Temporal Difference Model Predictive Control. Learns a latent world model with a value-equivalent objective, then uses MPPI for action selection. State-of-the-art on continuous control benchmarks.
Latent MPC MPPI Value-equiv
DIAMOND
Alonso et al., 2024 · NeurIPS
Uses a diffusion model as the world model to generate photorealistic imagined frames. Achieves human-level Atari from a diffusion-based environment model. Frontier of generative world models.
Diffusion WM Generative Expensive
DayDreamer
Wu et al., 2022
Adaptation of DreamerV2 for real physical robots. Demonstrates MBRL on quadruped locomotion, arm manipulation, and wheeled robots directly in the real world — not in simulation. Key robotics MBRL baseline.
Real Robot Dreamer-based Physical
LOOP / SLBO
Luo et al., 2018 · ICLR
Algorithmic Framework for MBRL with theoretical guarantees on policy improvement. Uses a monotonic improvement bound to safely blend real and model data.
Theoretical Monotonic
Dreaming Falcon
2025 · Quadrotor-specific
Physics-informed MBRL for quadcopters. Replaces the GRU world model with a physics-based 6-DOF RK4 integrator predicting forces/moments, improving accuracy for aerial dynamics.
Physics-informed Quadrotor RK4

Strengths & Weaknesses of MBRL

✦ Strengths
✓ Sample Efficiency. By generating thousands of synthetic rollouts from the learned model, MBRL can match or beat model-free algorithms with 10–100× less real environment data.
✓ Planning Capability. The model enables lookahead reasoning — the agent can mentally simulate consequences of actions before executing them.
✓ Transfer & Adaptation. A learned world model can be reused across tasks — only the reward head needs to change, not the full dynamics model.
✓ Uncertainty Quantification. Ensemble methods (PETS, MBPO) provide principled uncertainty estimates, enabling safe exploration.
✓ Safe RL Potential. The model can be used to predict and avoid dangerous states before they occur, critical for real robot deployment.
✦ Weaknesses
✗ Compounding Model Error. Small inaccuracies in the dynamics model accumulate over long rollouts, leading the policy to exploit model flaws ("model exploitation bias").
✗ Model Learning Complexity. Training a high-fidelity dynamics model is itself a hard supervised learning problem, adding significant algorithmic and computational overhead.
✗ Distribution Shift. The model is trained on observed states; if the policy visits new states, the model's predictions become unreliable.
✗ High Compute Cost. World models like DreamerV3 require expensive training (~240 hours for drone racing). Not practical for all research budgets.
✗ Stochastic Dynamics Challenge. Highly stochastic environments (turbulence, contact physics) are very hard to model accurately.

The Model Exploitation Problem — In Detail

This is MBRL's most dangerous failure mode. Because the policy is trained on imagined data, it can find spurious state-action combinations that look high-reward under the learned model but are actually nonsensical. The policy becomes overly optimistic and fails in the real environment.

⚡ MBPO's Solution: Short Rollout Horizon

MBPO showed empirically and theoretically that limiting imagined rollouts to H = 1 to 5 steps dramatically reduces compounding error while still providing significant sample efficiency gains. There is a sweet spot: too short and no benefit over model-free; too long and model error explodes.

// Policy improvement bound (MBPO, Janner et al. 2019) η[π] ≥ η̂_model[π] - C · ε_m / (1-γ)² // where: ε_m = max_t E_π[D_TV(p̂(s'|s,a) || p(s'|s,a))] // model error C = constant depending on rollout horizon H // longer H → larger C → looser bound → more risk
🧠 Nurturing Question

MBPO uses very short rollouts (H=1–5) to mitigate compounding error, while DreamerV3 uses H=15 imagination steps and works fine. What structural difference between MBPO's ensemble model and DreamerV3's RSSM might explain why DreamerV3 tolerates longer rollouts?

MBRL for Aerial Robotics — Your Research Domain

Quadrotors are a natural fit for MBRL: real-world data collection is expensive (hardware damage, resets), sample efficiency is critical, and the dynamics are sufficiently structured that a physics-informed model can be highly accurate. Here is the current landscape of MBRL applied to UAVs.

Why MBRL Fits Quadrotors Especially Well

Sample Cost
Real quadrotor data collection is slow and crash-prone. MBRL's 10–100× sample efficiency means fewer dangerous real-world rollouts are needed before the policy becomes competent.
Critical Benefit
Physics Priors
Quadrotor rigid-body dynamics are well-understood. Physics-informed world models (e.g. Dreaming Falcon's RK4 integrator) can embed Newton's laws, drastically reducing the data needed to fit the residual dynamics.
Inductive Bias
Partial Observability
Vision-based flight with a forward-facing camera is a POMDP — you can't fully observe the world state. RSSM-based world models explicitly handle partial observability through their recurrent deterministic state h_t.
POMDP
Fast Agile Maneuvers
Post-stall aerobatics and drone racing require planning over multiple steps. MBRL's lookahead capability is a direct advantage over reactive model-free policies.
Planning Horizon

Dream to Fly — DreamerV3 on Drone Racing (2025)

A 2025 paper demonstrated DreamerV3 training visuomotor policies for agile drone racing directly from pixels — no intermediate state representation, no imitation learning bootstrap. Key findings compared to model-free baselines (PPO, SAC):

Aspect DreamerV3 PPO (model-free) SAC (model-free)

Dreaming Falcon — Physics-Informed World Model (2025)

A critical insight for quadrotors: standard RNN-based world models (like DreamerV3's GRU) struggle with quadrotor dynamics because they don't respect the rigid-body physics structure. Dreaming Falcon replaces the recurrent transition model with a physics-based 6-DOF Runge-Kutta integrator:

// Dreaming Falcon world model // Instead of: h_t = GRU(h_{t-1}, z_{t-1}, a_{t-1}) // Use physics: [F_net, τ_net] = f_θ(h_t, a_t) // NN predicts forces/moments [pos, vel, att, ω] = RK4_integrate( // integrate Newton's laws state_t, F_net, τ_net, Δt ) // Benefits: // 1. Respects conservation laws by construction // 2. Better extrapolation to unseen maneuvers // 3. Physically interpretable

Open Research Challenges in Aerial MBRL

Aerodynamic Stochasticity
Turbulence, propeller wash, and ground effect introduce high stochasticity. Current RSSM models underfit this. Gaussian process or diffusion-based models may help.
Open Problem
Real Pixel Transfer
Dream to Fly used a hardware-in-the-loop setup with rendered pixels, not real camera images. The domain gap for real outdoor scenes remains an open problem.
Open Problem
Compute Budget
240 hours on a Quadro RTX 8000 is prohibitive for most research labs. Smaller world model architectures optimized for aerial tasks are needed.
Scalability
Safe Exploration
MBRL should enable safe exploration via model-predicted constraint checking, but integrating hard safety constraints (obstacle avoidance) into the imagination loop is still largely unsolved.
Safety
🚁 Recommended Starting Point for Your Research

For a quadrotor MBRL project: start with DreamerV3 + Flightmare/Isaac Lab for simulation. Use the physics-informed modification from Dreaming Falcon as a model architecture improvement. Ground your experiments in the state-vector observation space first (not pixels) to isolate dynamics modeling from perception. Then add pixel observations once the state-based policy is solid — matching the progression in "Dream to Fly."

🧠 Nurturing Question

Dreaming Falcon embeds a 6-DOF RK4 integrator as the world model's transition backbone. This gives the model strong inductive bias from physics. But quadrotors also have complex rotor aerodynamics, blade flapping, and motor delays that the simplified rigid-body model ignores. How would you design the neural network component f_θ to best capture these residual dynamics — and what observations would you include as input to f_θ?
