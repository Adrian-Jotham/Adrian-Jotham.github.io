§ 4.1

Introduction to Dynamic Programming

Dynamic Programming (DP) is a mathematical strategy for solving complex problems by breaking them into smaller, overlapping subproblems, solving each once, and storing the results for reuse. This avoids redundant computation and is the foundation of most RL planning algorithms.

🧩

Optimal Substructure

The solution to a larger problem can be built from optimal solutions to its sub-problems. Think: the shortest path from A→C passes through the shortest path A→B.

🗂️

Memoization

Store solutions to subproblems so they're never recomputed. This transforms exponential complexity into polynomial time in many problems.

🔁

Iterative Refinement

DP in RL works by repeatedly applying update rules (Bellman equations) to estimates of state values until they converge to the true optimal values.

In RL, DP requires full knowledge of the environment — transition probabilities P(s'|s,a) and rewards R(s,a). This distinguishes it from Monte Carlo (needs complete episodes) and TD-learning (incremental, no model needed).

// Bellman Optimality Equation
V*(s) = max_a [ R(s,a) + γ · Σ P(s'|s,a) · V*(s') ]
V*(s) — optimal value of state s R(s,a) — immediate reward for action a in state s γ      — discount factor (0 ≤ γ < 1) P(s'|s,a) — transition probability to next state s'
💡 Analogy

Planning a road trip: to find the optimal overall route, you break the journey into legs and solve each leg optimally. The full optimal path is assembled from optimal sub-paths.

§ 4.2

Policy Evaluation & Improvement

These two steps are the core engine of Policy Iteration — the first fundamental DP algorithm for solving MDPs. They alternate until convergence to an optimal policy.

1

POLICY EVALUATION — Compute V^π(s)

Given a fixed policy π, iteratively apply the Bellman Expectation Equation to compute the expected cumulative reward from each state. Repeat until updates are smaller than threshold θ.

2

POLICY IMPROVEMENT — Greedy Update

Using the computed V^π(s), generate a better policy π' by selecting the action that maximizes expected return: π'(s) = argmax_a [ R(s,a) + γ Σ P(s'|s,a) V^π(s') ]

3

REPEAT UNTIL STABLE

If π' = π, the policy is optimal and the algorithm stops. Convergence is guaranteed by the contraction mapping property of the Bellman operator.

// Bellman Expectation Equation (for evaluation)
V^π(s) = E[ R(s,a) + γ · V^π(s') ] where a ~ π, s' ~ P
💡 Analogy

Navigating with a map: once you know the distances from each location (evaluation), you always pick the shortest next step (improvement). You keep refining your plan until it stops changing.

// IMPLEMENTATION NOTE The Grid World example in the chapter uses a 4×4 grid with a goal at state 15. Each step incurs a −0.1 penalty, the goal gives +1.0 reward, and γ = 0.95. The optimal policy guides the agent to the goal via the shortest path.
§ 4.3

Value Iteration

Value Iteration is a more direct DP algorithm that skips separate policy evaluation and instead applies the Bellman Optimality Equation directly and repeatedly until the value function converges — then extracts the optimal policy in one final step.

// Value Iteration Update Rule
V_{k+1}(s) = max_a [ R(s,a) + γ · Σ P(s'|s,a) · V_k(s') ]
Iterates until: max_s |V_{k+1}(s) − V_k(s)| < θ

The key difference from Policy Iteration is that there is no separate inner loop for policy evaluation. Each sweep of all states performs one Bellman optimality update, combining evaluation and improvement.

Property Policy Iteration Value Iteration
Structure Two nested loops (eval + improve) One loop (combined update)
Convergence Fewer outer iterations More sweeps, simpler per-step
Equation Used Bellman Expectation Bellman Optimality
Policy extraction Maintained throughout Extracted at the end
Computational cost Higher per outer iteration Lower per sweep
Best for Problems needing fewer policy updates Problems needing fast convergence
// CONVERGENCE GUARANTEE With finite state/action spaces and γ < 1, the Bellman operator is a contraction mapping. This mathematically guarantees both algorithms converge to the unique optimal value function V*.
💡 Analogy

Career planning: initially you estimate the value of different career paths. Over time, you gather more information and keep refining your estimates until they stabilize into your optimal career choice — that stabilization point is V*.

§ 4.4

Advanced DP Techniques

Standard DP updates all states synchronously in every iteration — expensive for large state spaces. Two advanced techniques improve efficiency significantly:

⚡ Asynchronous DP

Update only a subset of states S' ⊆ S each iteration instead of all states. Updates are applied immediately, so other states benefit from the freshest estimates. Convergence is still guaranteed with appropriate state selection.

🎯 State Prioritization

Dynamically decide which states to update by measuring the Bellman residual |V_{k+1}(s) − V_k(s)|. States with high residuals are updated first — they are the most "unsettled" and offer the most improvement.

🔃 Prioritized Sweeping

Combines asynchronous updates with a priority queue. After updating a state, its predecessors are added to the queue if their residuals are large. Focuses computation on high-impact areas like goal states or obstacles.

📐 Bellman Residual

The metric that drives prioritization: Residual(s) = |V_{k+1}(s) − V_k(s)|. Large residual = the value estimate here is still changing a lot = worth updating. Small residual = already converged, skip for now.

// Asynchronous Update Rule
V_{k+1}(s) = Bellman(s) if s ∈ S' (prioritized set) V_{k+1}(s) = V_k(s) otherwise
💡 Analogy — Road Trip Planning

Synchronous DP recalculates every route in your region each day. Asynchronous DP with prioritization only recalculates routes where traffic patterns changed significantly — saving effort and focusing on what matters most.

Real-world applications include robot navigation (prioritize states near obstacles), autonomous vehicles (focus at intersections), and game AI (focus on high-strategy states). These techniques make DP practical at scale.

§ 4.5

Key Takeaways

DP requires a full model

Unlike TD or Monte Carlo, DP needs transition probabilities and rewards — ideal for structured, well-defined environments.

Two core equations

Bellman Expectation (evaluate a policy) vs. Bellman Optimality (find the best policy). Both are iteratively solved.

γ controls horizon

The discount factor balances short vs. long-term rewards. Near 1 = patient, long-sighted agent. Near 0 = greedy, short-sighted agent.

Convergence is guaranteed

For finite MDPs with γ < 1, both Policy and Value Iteration converge to the unique optimal policy V* and π*.

🎓 Follow-Up Questions to Deepen Your Understanding

Think through these before moving to Chapter 5:

  • Why does DP need a complete model of the environment? What would happen if the transition probabilities were unknown? Which algorithm family would you use instead?
  • Policy Iteration vs. Value Iteration: Both converge to V* — but which would you prefer if the state space is very large? Why? What is the trade-off with the discount factor γ?
  • On the Bellman residual: If a state has a very small residual, does that mean its value is correct? Or could it just mean the values of its neighbors haven't changed yet?
  • Asynchronous DP: Could asynchronous updates ever slow down convergence compared to synchronous updates? Under what conditions might that happen?
  • Connecting to the real world: The Grid World uses discrete states and actions. How would you handle a robot with continuous position (x,y)? What breaks, and how is this addressed in Part IV of the book?
